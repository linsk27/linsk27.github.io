Dev Cockpit 资源雷达工作台

Dev Cockpit 是 linsk27 的本地资源雷达工作台，用来收集 GitHub 仓库、Demo、工具、教程和 AI coding 资源，并通过卡片库、分类筛选、价值评分和星云图整理开发资料。

Dev Cockpit 现在更像一个本地资源雷达，而不是普通的项目启动器。它把我平时看到的 GitHub 仓库、Demo、工具、教程、视频号文本和 AI coding 资源收进一个工作台，再用分类、标签、摘要、价值评分和可视化视图帮助我判断哪些东西值得继续研究。

为什么做这个

开发过程中真正麻烦的不是找不到资料，而是资料太分散。一个工具可能来自 GitHub，一个设计技能来自别人分享的 Skill.md，一个工作流来自社区仓库，一个教程来自短文本记录。看过之后如果没有及时归档，很快就会忘掉它为什么有价值。

Dev Cockpit 的目标是把这些资源变成可以复盘的卡片，而不是只留一串链接。每条资源都会保留来源、摘要、类型、子分类、状态和评分，后续可以按关键词、标签、来源和大类快速筛选。

当前资源页

资源页现在有两个主要视图。

卡片库适合做日常整理。页面顶部可以粘贴 GitHub、Demo、工具、教程或视频号文本，也可以导入、导出和解析资源。中间区域提供搜索框、状态筛选和大类筛选，目前支持工具、Skills、Workflow 等分类。

星云图适合看资源之间的分布。它把资源按照大类和主题放进一个可视化空间里，适合快速判断当前收集内容偏向前端工具、AI 编程辅助、视觉设计、预测模拟还是 3D 生成。

它解决什么问题

  • 把零散链接整理成有摘要、有标签、有来源的资源卡片。
  • 区分工具、Skills、Workflow 等不同类型，避免所有资料混在一起。
  • 给资源打价值评分，帮助优先看更值得研究的内容。
  • 支持搜索、状态筛选和大类筛选，让资源库不会越积越乱。
  • 通过星云图观察资源分布，快速发现自己最近关注的技术方向。
  • 保持本地运行，适合个人开发者维护自己的资料库。

现在收集的资源样例

这版截图里可以看到 taste-skill、everything-claude-code、FreeDomain、react-bits、MiroFish、andrej-karpathy-skill、articraft、huashu-design 等资源。它们覆盖前端设计技能、AI 编程工作流、免费域名、React 动效组件、群体智能预测、3D 资产生成和智能体设计生成等方向。

这些资源不是简单展示名称，而是会被整理成摘要、来源、分类、标签和评分。这样后面回看时，不需要重新打开每个仓库才能想起来它适合什么场景。

适合谁

  • 经常收藏 GitHub 仓库、工具、教程和 AI coding 资源的人。
  • 想把资源从聊天记录、浏览器收藏夹和临时文档里整理出来的人。
  • 需要给资源做分类、评分、状态管理和后续复盘的人。
  • 正在研究 AI coding workflow、Agent Skills、前端工具链和设计生成工具的人。
  • 想参考 Electron、Vue、TypeScript 本地工作台界面的人。
